# Vibe Coding 核心原则

## 1. 核心理念

### 1.1 规划就是一切
- **谨慎让 AI 自主规划**，否则代码库会变成一团无法管理的乱麻
- **先结构，后代码**，一定要规划好框架，不然后面技术债还不完
- **目的主导**：开发过程中的一切动作围绕"目的"展开

### 1.2 上下文是第一性要素
- **垃圾进，垃圾出**，高质量的上下文是获得高质量输出的关键
- **系统性思考**，从实体、链接、功能/目的三个维度考虑问题
- **数据与函数即是编程的一切**

### 1.3 凡是 AI 能做的，就不要人工做
- **一切问题问 AI**
- **输入，处理，输出**刻画整个过程
- **多问 AI 是什么？为什么？怎么做？**

## 2. 工作流程

### 2.1 道（理念）
- **奥卡姆剃刀定理**，如无必要，勿增代码
- **帕累托法则**，关注重要的那20%
- **逆向思考**，先明确需求，从需求逆向构建代码
- **重复，多试几次**，实在不行重新开个窗口
- **专注**，极致的专注可以击穿代码，一次只做一件事

### 2.2 法（方法）
- **一句话目标 + 非目标**
- **正交性**，功能不要太重复
- **能抄不写**，不重复造轮子，先问 AI 有没有合适的仓库
- **一定要看官方文档**，先把官方文档爬下来喂给 AI
- **按职责拆模块**
- **接口先行，实现后补**
- **一次只改一个模块**
- **文档即上下文**，不是事后补

### 2.3 核心方法框架

#### 2.3.1 反推法：从交付物入手
- **定义**：从最终的交付物出发，反向推导所需的步骤和资源
- **应用场景**：
  - 代码开发：从预期的代码结构和功能反向推导实现步骤
  - 文档编写：从最终文档的结构和内容反向推导所需的信息和组织方式
  - 测试设计：从预期的测试结果反向推导测试用例和测试策略
- **实施步骤**：
  1. 明确最终交付物的具体要求和标准
  2. 分解交付物为可管理的组件和模块
  3. 为每个组件制定详细的实现计划
  4. 按计划执行并验证每个组件的质量

#### 2.3.2 归纳法：从聊天记录入手
- **定义**：从用户的聊天记录和交互历史中提取模式和需求
- **应用场景**：
  - 用户需求分析：从用户的问题和反馈中提取核心需求
  - 问题诊断：从错误信息和用户描述中识别问题根源
  - 功能优化：从用户的使用习惯和反馈中发现改进机会
- **实施步骤**：
  1. 收集和整理用户的聊天记录和交互历史
  2. 分析记录中的模式、关键词和情感
  3. 提取核心需求和问题点
  4. 基于分析结果制定解决方案

#### 2.3.3 流程法：从工作步骤入手
- **定义**：通过标准化的工作流程来确保任务的一致性和质量
- **应用场景**：
  - 开发流程：标准化的代码开发、测试和部署流程
  - 审核流程：标准化的代码审查和质量检查流程
  - 协作流程：标准化的团队协作和沟通流程
- **实施步骤**：
  1. 定义清晰的工作流程和步骤
  2. 为每个步骤制定明确的输入、输出和质量标准
  3. 实施流程并监控执行情况
  4. 持续优化流程以提高效率和质量

#### 2.3.4 经验提炼法：从隐性知识入手
- **定义**：从经验和隐性知识中提取可复用的模式和最佳实践
- **应用场景**：
  - 知识管理：将个人经验转化为团队共享的知识
  - 问题解决：利用历史经验快速解决类似问题
  - 技能传递：将专家知识传递给新团队成员
- **实施步骤**：
  1. 识别和收集隐性知识和经验
  2. 分析和提炼可复用的模式和原则
  3. 将隐性知识转化为结构化的文档和指南
  4. 建立知识共享和更新机制

### 2.4 术（技巧）
- 明确写清：**能改什么，不能改什么**
- Debug 只给：**预期 vs 实际 + 最小复现**
- 测试可交给 AI，**断言人审**
- 代码一多就**切会话**

### 2.5 器（工具）
- **IDE**：Visual Studio Code、Cursor、Neovim (LazyVim)
- **终端**：Warp、tmux
- **AI 模型**：Claude Opus 4.5、gpt-5.1-codex、Gemini 3.0 Pro
- **辅助工具**：Mermaid Chart、NotebookLM、Zread

## 3. 实施流程

### 3.1 规划阶段
1. **定义目标**：明确项目的核心目标和非目标
2. **技术选型**：选择合适的技术栈
3. **架构设计**：设计系统架构和模块划分
4. **实施计划**：生成详细的分步实施计划

### 3.2 执行阶段
1. **上下文准备**：创建记忆库，收集相关文档
2. **小步迭代**：按计划分步实现，每步都有测试
3. **代码审查**：AI 自动审查代码，人类审核
4. **测试验证**：AI 生成测试用例，人类验证

### 3.3 优化阶段
1. **性能优化**：识别并优化性能瓶颈
2. **代码重构**：优化代码结构和可读性
3. **文档完善**：更新文档，确保与代码一致
4. **经验沉淀**：记录经验教训，持续改进

## 4. 最佳实践

### 4.1 提示词设计
- **清晰明确**：具体说明需求和期望
- **上下文丰富**：提供足够的背景信息
- **约束边界**：明确能做什么，不能做什么
- **迭代优化**：根据反馈不断优化提示词

### 4.2 代码管理
- **模块化设计**：按功能划分模块，职责清晰
- **版本控制**：使用 Git 进行版本管理
- **测试驱动**：先写测试，再实现功能
- **代码规范**：遵循统一的代码规范

### 4.3 团队协作
- **文档共享**：使用共享文档记录决策和进展
- **代码审查**：定期进行代码审查
- **知识传递**：记录经验和最佳实践
- **持续集成**：自动化测试和部署

## 5. 注意事项

### 5.1 风险防范
- **代码质量**：不要盲目接受 AI 生成的所有代码
- **安全隐患**：注意代码中的安全漏洞
- **依赖管理**：合理管理依赖，避免版本冲突
- **性能问题**：关注代码性能，避免不必要的开销

### 5.2 持续改进
- **反馈循环**：建立反馈机制，持续优化
- **学习总结**：定期总结经验，改进流程
- **工具更新**：关注工具和模型的更新
- **技能提升**：不断学习新的技术和方法

## 6. 结语

Vibe Coding 不是一款工具，而是一种编程范式的革命。它强调规划驱动、上下文固定和模块化执行，将编程从"人类向机器翻译需求"的过程，转变为"人类向 AI 表达意图，AI 完成技术落地"的人机协作范式。

通过遵循这些核心原则，开发者可以充分发挥 AI 的能力，同时保持对项目的控制和理解，实现从想法到可维护代码的高效转化。
