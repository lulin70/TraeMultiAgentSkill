# 多角色代码走读 Prompt 模板

## 概述

本目录包含用于多角色代码分析的 prompt 模板。每个角色（架构师、产品经理、独立开发者、UI设计师、测试专家）都有专属的分析视角和输出格式。

## 角色列表

| 角色 | Prompt 模板 | 关注点 |
|------|-------------|--------|
| 架构师 | [architect-code-analysis.md](architect-code-analysis.md) | 系统架构、模块划分、技术选型 |
| 产品经理 | [pm-code-analysis.md](pm-code-analysis.md) | 业务功能、用户流程、需求完整性 |
| 独立开发者 | [coder-code-analysis.md](coder-code-analysis.md) | 代码实现、函数逻辑、错误处理 |
| UI设计师 | [ui-code-analysis.md](ui-code-analysis.md) | 界面组件、交互流程、用户体验 |
| 测试专家 | [test-code-analysis.md](test-code-analysis.md) | 测试覆盖、边界条件、质量风险 |

## 使用方法

### 在 Agent 流程中使用

当需要多角色分析时，按照以下流程调用：

```
1. 收集项目基本信息（项目扫描）
2. 分发各角色分析任务（并行执行）
3. 收集各角色的分析结果
4. 执行文档对齐
5. 生成统一代码地图
```

### 角色分析模板使用示例

```markdown
## 角色分析任务

请从 [角色名称] 的视角分析以下代码：

**项目信息**:
- 项目名称: {project_name}
- 工作空间: {workspace_name}
- 编程语言: {languages}

**代码文件列表**:
{file_list}

**请按照 {role_name} 的视角分析以下内容**:
{focused_questions}

**输出格式**:
按照 {role_template_path} 中的格式输出分析结果。
```

## 文档对齐原则

1. **共识优先**: 多个角色共同确认的信息优先保留
2. **差异标注**: 不同视角产生的差异需要明确标注
3. **互补增强**: 不同角色的分析可以互补，增强理解
4. **冲突解决**: 当冲突发生时，按照以下优先级解决：
   - 架构师视角 > 产品经理视角 > 独立开发者视角
   - 代码证据 > 推测分析
