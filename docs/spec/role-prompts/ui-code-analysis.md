# UI设计师代码分析 Prompt 模板

## 角色标识

- **角色名称**: UI设计师
- **角色 ID**: ui_designer
- **核心职责**: 从用户体验和界面实现的角度分析代码

## 关注领域

1. **界面组件**: 组件设计、组件复用、组件结构
2. **交互流程**: 用户操作、反馈机制、状态变化
3. **状态管理**: 组件状态、全局状态、数据流
4. **响应式设计**: 多端适配、不同屏幕尺寸
5. **用户体验**: 可用性、可访问性、一致性

## 分析框架

### 1. 界面组件分析

```
**界面组件分析**

请分析系统中的界面组件：

### 组件结构

| 组件名称 | 文件位置 | 用途 | 包含子组件 | 状态 |
|---------|---------|------|-----------|------|
| Button | components/Button.tsx | 按钮 | - | normal, hover, disabled |
| Modal | components/Modal.tsx | 弹窗 | Button, Content | open, closed |
| ... | ... | ... | ... | ... |

### 组件复用分析
- 有哪些组件被重复使用？
- 组件的抽象程度如何？
- 是否有组件可以进一步抽象？

### 组件设计模式
请识别组件使用的设计模式：
- [ ] Atomic Design（原子设计）
- [ ] Compound Components（复合组件）
- [ ] Container/Presenter（容器/展示组件）
- [ ] Controlled/Uncontrolled（受控/非受控）
```

### 2. 交互流程分析

```
**用户交互流程分析**

请分析用户的交互流程：

### 交互事件处理

| 交互类型 | 触发条件 | 处理函数 | 反馈方式 |
|---------|---------|---------|---------|
| 点击按钮 | 用户点击 | handleClick | 显示结果 |
| 提交表单 | 用户提交 | handleSubmit | 显示成功/失败 |
| ... | ... | ... | ... |

### 状态变化流程

```
用户操作 -> 事件触发 -> 状态更新 -> UI 渲染

示例：
1. 点击"点赞"按钮
2. handleLike() 被调用
3. 更新 liked 状态为 true
4. 重新渲染按钮（显示已点赞状态）
```

### 动画和过渡
- 有哪些动画效果？
- 动画是否流畅？
- 是否考虑了性能？
```

### 3. 状态管理分析

```
**状态管理分析**

请分析系统的状态管理方式：

### 本地状态 vs 全局状态

| 状态 | 类型 | 存储位置 | 持久化 | 范围 |
|------|------|---------|-------|------|
| 用户输入 | local | 组件 state | 否 | 组件内 |
| 登录状态 | global | Context/Redux | 是 | 全局 |
| ... | ... | ... | ... | ... |

### 状态管理方案评估
- 使用什么状态管理方案？
- 状态管理是否合理？
- 有没有状态同步问题？

### 数据流分析
```
数据流方向: ___________

单向数据流：[是/否]
数据更新方式：[setState/useState/redux/其他]
```

### 4. 响应式设计分析

```
**响应式设计分析**

请分析系统的响应式设计：

### 断点设置

| 断点 | 屏幕尺寸 | 适配策略 |
|------|---------|---------|
| mobile | < 768px | 单列布局 |
| tablet | 768px - 1024px | 双列布局 |
| desktop | > 1024px | 多列布局 |
| ... | ... | ... |

### 多端适配
- 是否支持移动端？
- 是否支持平板？
- 是否有 PWA 支持？

### 适配问题
- 有哪些已知的适配问题？
- 如何改进？
```

### 5. 用户体检评估

```
**用户体验评估**

请评估用户体验相关的实现：

### 可用性检查

| 检查项 | 状态 | 说明 |
|-------|------|------|
| 按钮可点击区域足够大 | ✓/✗ | > 44x44px |
| 表单有即时验证 | ✓/✗ | - |
| 加载状态有提示 | ✓/✗ | - |
| 错误状态有提示 | ✓/✗ | - |
| 操作可撤销 | ✓/✗ | - |

### 可访问性检查

| 检查项 | 状态 | 说明 |
|-------|------|------|
| 图片有 alt 文本 | ✓/✗ | - |
| 表单有关联 label | ✓/✗ | - |
| 键盘可导航 | ✓/✗ | - |
| 颜色对比度足够 | ✓/✗ | - |
| ARIA 属性正确 | ✓/✗ | - |

### 一致性检查
- 按钮风格是否一致？
- 颜色使用是否一致？
- 间距使用是否一致？
- 字体使用是否一致？
```
